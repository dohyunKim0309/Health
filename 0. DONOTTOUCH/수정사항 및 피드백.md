# Opus 4.7 피드백
### 1. Caveats "None"의 정직성 불가능

현 67페이지 모두 YouTube 전사 기반이고 각 소스는 수많은 세부를 담은 전체 강의다. 정직하게 채우면 대부분 페이지에 "투여 세부·부작용 프로파일·아형·정량 효과 크기 미포함" 같은 caveat가 들어가 **필드가 노이즈가 된다**. 정직한 "None"은 거의 불가능.

**제안**: 지침에 범위 한정을 명시 — "Caveats는 **소스가 다루지만 이 페이지가 단순화한 주제**만. 소스가 안 다루는 것은 caveat가 아니라 범위 밖." + 예시로 구체적 항목만 허용("청소년 복용량·임신 데이터"), 막연한 범주어 금지("여러 아형").

### 2. Gap 2 트리거의 날카로움이 Caveats 품질에 전적으로 의존

"새 자료가 Caveats 주제를 건드림 → raw 조회"는 Caveats가 구체적일 때만 정확히 발화한다. 막연하면 매번 발화 → 사실상 raw를 매번 읽게 된다. 1번 문제와 연결된다.

### 3. 마이그레이션 경로 미정의 (가장 실무적 걱정)

현 위키는 Caveats 없음·태그 없음. 새 지침의 모든 트리거가 **기존 페이지에서는 발화하지 못한다**. 세 옵션:

- (a) **일괄 백필** — 67페이지 × 각 원 소스 재방문. 큰 작업이지만 일관성 확보.
- (b) **접촉 시 점진 마이그레이션** — 건드리는 페이지만 업데이트. 긴 기간 혼합 상태.
- (c) **중간 상태 규칙** — "Caveats 없는 기존 페이지는 모호 상태로 간주, 신규 자료가 해당 페이지 주제를 건드리면 raw 참조 기본값" — 자연스럽지만 지침에 명시해야 워크플로우가 맞음.

**제안**: 지침에 한 문장으로 전략 결정 명시.

### 4. 태그 어휘 드리프트 위험

67페이지 동시 부여 시 시드 어휘 없으면 `#ad` vs `#atopic-dermatitis` vs `#eczema`, `#adhd` vs `#attention-deficit`, `#sleep` vs `#sleep-disorder` 등 중복 태그 난립 확실. "기존 태그 재사용"은 옳으나 **시작점이 있어야** 재사용이 의미 있다.

**제안**: 부여 시작 전 **기본 태그 팔레트** 30–50개를 `2. WIKI/tags.md`로 문서화 후 진행. 또는 첫 스캔에서 자동 추출 → 사용자 승인 → 고정.

### 5. 한글 태그 정책 부재

현 위키는 3층 한글 스캐폴딩 + Aliases 필드에 한글 별칭. 태그에도 `#아토피` 같은 한글 태그를 허용하면 Aliases와 **중복**이다.

**제안**: 태그는 **영어 전용**, 한글 매칭 서피스는 Aliases에 집중 — 지침에 명시.

### 권고

채택하되 지침 초안에 **3줄 추가**:

1. Caveats 범위 한정 ("소스가 다루되 이 페이지가 단순화한 것만")
2. 마이그레이션 전략 선택 (나는 **중간 상태 규칙(3c)**를 추천 — 가장 적은 초기 비용)
3. 태그 팔레트 위치 + 한글 태그 정책(영어 전용)

이 세 개만 해결되면 나머지는 시행 중 조정 가능한 수준이다. 가장 큰 실질 위험은 **Caveats 필드가 형식만 있고 실질은 비어 있는 상태** — 지침의 핵심 메커니즘(Gap 2 트리거)이 무력화된다. Caveats 정의를 엄격히 잡는 것이 가장 중요하다.
